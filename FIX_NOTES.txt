EDGE REQUESTS FIX — topsarkarijobs.com (Vercel 3.3M/1M paused)
=============================================================

ROOT CAUSE
----------
1. vercel.json was EMPTY ({"version":2}) — Vercel IGNORES the _headers file
   (that's Netlify format). So NO cache rules were applied on Vercel.
   Every request hit the edge = millions of edge requests.
2. version.json polled every 5 min per tab WITH ?_t= cache-buster (CDN skip).
3. cache-manager.js fired background refresh every 5 min + on EVERY tab focus.
4. script.js fetched header.html/footer.html/config.json with no-store on
   EVERY page load (3 forced edge requests x 3245 pages).

CHANGES
-------
vercel.json     : NEW — real Vercel headers block. JSON/HTML/JS/CSS/images now
                  CDN-cached with s-maxage + stale-while-revalidate. THIS is the
                  main fix — without it nothing else was being cached on Vercel.
tsj-version.js  : poll 5 min -> 6 hours; removed ?_t= cache-buster; no-store->no-cache.
cache-manager.js: background 5-min refresh DISABLED; refresh-on-every-focus removed
                  (now only if data >6h old); interval 5min -> 6h.
script.js       : header/footer/config fetches no-store -> default (cacheable).
script.min.js   : same 3 fetches patched (site loads .min, not source).
sw.js           : version.json served via CDN (removed forced no-store).
_headers        : Netlify fallback updated to match (version/config/tsj-version).

DEPLOY
------
Replace these 7 files in repo root, commit, push. Vercel will redeploy.
Expected: edge requests drop ~80-95%. Resume service / wait for cycle reset.
Note: visible UI/layout unchanged.
